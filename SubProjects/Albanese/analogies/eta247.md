## Decisions identified

### Decision: do the four named glue lemmas exist with usable signatures?

- **Mathlib idiom**: all three categorical glue lemmas exist verbatim in
  `Mathlib.CategoryTheory.Adjunction.Unique`:
  - `CategoryTheory.Adjunction.homEquiv_leftAdjointUniq_hom_app` —
    `(adj1.homEquiv x (F'.obj x)) ((adj1.leftAdjointUniq adj2).hom.app x) = adj2.unit.app x`.
  - `CategoryTheory.Adjunction.unit_leftAdjointUniq_hom_app` —
    `adj1.unit.app x ≫ G.map ((adj1.leftAdjointUniq adj2).hom.app x) = adj2.unit.app x`.
  - `CategoryTheory.Adjunction.leftAdjointUniq_hom_app_counit` —
    `(adj1.leftAdjointUniq adj2).hom.app (G.obj x) ≫ adj2.counit.app x = adj1.counit.app x`.
  Re-association variants also present: `unit_leftAdjointUniq_hom_app_assoc`,
  `leftAdjointUniq_hom_app_counit_assoc`, `leftAdjointUniq_trans_app_assoc` (same module).
  The fourth item — `Functor.LaxMonoidal.ε (SheafOfModules.pushforward φ) = unitToPushforwardObjUnit φ`
  by `rfl` — is a project-side defeq claim (prover-verified), not a Mathlib lemma.
- **Project's current path**: the prover plans to drive the chase with exactly these names.
- **Gap**: identical — names + signatures match the prover's plan; no renames at the pinned commit.
- **Verdict**: PROCEED.

### Decision: is the two-level leftAdjointUniq mate chase closeable with standard API, or is a coherence lemma missing?

- **Mathlib idiom**: the reduction lemmas the prover transposes through all exist:
  - `CategoryTheory.Adjunction.homEquiv_unit` / `homEquiv_counit`
    (`Mathlib.CategoryTheory.Adjunction.Basic`).
  - `CategoryTheory.Adjunction.leftAdjointOplaxMonoidal_η`
    (`Mathlib.CategoryTheory.Monoidal.Functor`):
    `Functor.OplaxMonoidal.η F = (adj.homEquiv _ _).symm (Functor.LaxMonoidal.ε G)` — this is the
    lemma that turns the η-bridge into the pushforward-side ε identity, i.e. it is *why* the mate
    chase is the canonical route.
  Plus `NatTrans` naturality and the project's own
  `pullbackPushforwardAdjunction_homEquiv_pullbackObjUnitToUnit`.
- **Project's current path**: the η-bridge is the UNIT-side mirror of `pullbackObjUnitToUnit_comp`
  (L904–990), which closed the COUNIT/δ-side mate transport axiom-clean in the same file using
  exactly this machinery (`homEquiv_unit`, `homEquiv_naturality_{left,right}`, `comp_unit_app`,
  pushforward-coherence naturality, all re-associated with `erw`). The only new ingredient is
  pushing `pullbackValIso.inv = (sheafificationCompPullback).hom = leftAdjointUniq.hom` (post-
  composed with a counit `mapIso`) through the chase — which is exactly what the three
  `leftAdjointUniq_*_app` glue lemmas are for.
- **Gap**: identical — no missing coherence lemma. The two-level chase across the presheaf↔sheaf
  sheafification boundary is closeable with the standard `Adjunction`/mate API; the `_assoc`
  variants supply the needed re-association under longer composites.
- **Cost / friction (not a gap)**: the same defeq-but-not-syntactic friction `pullbackObjUnitToUnit_comp`
  already absorbed with `erw` (`Scheme.Modules.pullback f` vs `SheafOfModules.pullback f.toRingCatSheafHom`,
  and here additionally `sheafificationCompPullback` unfolding to `leftAdjointUniq`). Elaboration
  nuisance, not an API absence.
- **Verdict**: PROCEED.

### Decision: is there a SHORTER idiom that sidesteps the manual mate chase entirely?

- **Candidate**: `CategoryTheory.Functor.Monoidal.instIsIsoη`
  (`Mathlib.CategoryTheory.Monoidal.Functor`): `[F.Monoidal] → IsIso (Functor.OplaxMonoidal.η F)`.
- **Why it does NOT apply**: it requires `F` to be STRONG monoidal (`F.Monoidal`), and (a) the goal
  is `a_Y.map (η F)` — sheafification applied to the unit of a *different* functor's oplax structure,
  not `η` of a single monoidal functor; (b) `F = PresheafOfModules.pullback φ'` is only oplax, not
  strong, monoidal — supplying that strong structure on the composite is precisely the general build
  the project deliberately abandoned (see [[pullback-tensor-loctriv-pivot]]). No
  `leftAdjointUniq` monoidal-compatibility lemma exists either.
- **Gap**: NEEDS_MATHLIB_GAP_FILL would be the verdict for a strong-monoidal shortcut — but the
  project should NOT pursue it; the mate chase via `leftAdjointOplaxMonoidal_η` IS the canonical and
  shortest available route.
- **Verdict**: PROCEED (no shorter idiom; the planned chase is already the idiom).

## Recommendation
PROCEED with the planned mate-calculus chase. Every lemma the prover named exists at the pinned
commit with the exact signatures required (three `leftAdjointUniq_*_app` glue lemmas + their `_assoc`
variants in `Mathlib.CategoryTheory.Adjunction.Unique`; `leftAdjointOplaxMonoidal_η`, `homEquiv_unit`,
`homEquiv_counit` in the monoidal/basic adjunction files). The chase is the unit-side mirror of the
already-axiom-clean `pullbackObjUnitToUnit_comp`, so the technique is proven to work in this exact
file and setting. There is no missing coherence lemma and no shorter monoidal-transport shortcut
(`instIsIsoη` needs the strong-monoidal structure the project intentionally does not build). Expect
the same `erw`-driven defeq friction as the sibling lemma; budget for it but do not treat it as a gap.
