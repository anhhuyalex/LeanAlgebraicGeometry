## Decisions identified

### Decision 1: Does the sheaf-level template exist as claimed, and is it a `pushforward ⊣ pushforward` adjunction?

- **Mathlib idiom**: YES. `SheafOfModules.pushforwardPushforwardAdj`
  (`Mathlib/Algebra/Category/ModuleCat/Sheaf/PushforwardContinuous.lean:226`) is a genuine
  `pushforward.{v} φ ⊣ pushforward.{v} ψ`, built from `adj : F ⊣ G` (underlying functors) plus two
  compatibility hypotheses `H₁`/`H₂` (lines 219-221). Its unit/counit are assembled from the
  sheaf-level helpers `pushforwardId` (`:67`), `pushforwardComp` (`:101`), `pushforwardCongr`
  (`:73`), `pushforwardNatTrans` (`:154`). The triangle identities (lines 235-247) are proved
  purely sectionwise (`ext U x` + `X.val.presheaf.map`, `← Functor.map_comp`, `← op_comp`,
  `adj.left/right_triangle_components`) — no sheaf-specific machinery.
- **Note**: the sheaf template itself does NOT use `leftAdjointUniq`. The consumer
  `AlgebraicGeometry/Modules/Sheaf.lean:340` (`Scheme.Modules.restrictAdjunction`) calls
  `pushforwardPushforwardAdj (f.isOpenEmbedding.isOpenMap.adjunction) _ _ ?_ ?_` directly to make
  `restrictFunctor f ⊣ pushforward f`, discharging `H₁`/`H₂` for the open-immersion case in 6 lines
  (`:346-353`) via `f.appIso`. **Those exact `H₁`/`H₂` proofs port to the presheaf level unchanged.**
- **Gap**: identical (the template is real and reusable as a blueprint).
- **Verdict**: PROCEED.

### Decision 2: Presheaf-level pushforward/pullback API — what exists, what is absent

- `PresheafOfModules.pushforward` (along `F` + `φ : S ⟶ F.op ⋙ R`): **EXISTS**,
  `Presheaf/Pushforward.lean:86` (`= pushforward₀ F R ⋙ restrictScalars φ`).
- `PresheafOfModules.pullback`: **EXISTS**, `Presheaf/Pullback.lean:44`, defined as
  `(pushforward φ).leftAdjoint` — the **opaque abstract left adjoint**. No sectionwise formula
  (confirms the project's iter-208 correction). The instance `(pushforward φ).IsRightAdjoint` is a
  genuine *theorem* (`:97-98`, via `isRightAdjoint_of_leftAdjointObjIsDefined_eq_top`), so `pullback`
  really exists, not hypothetically.
- `pushforwardId`, `pushforwardComp`: **EXIST** at presheaf level (`Pushforward.lean:125, 135`),
  plus `pushforward_assoc/comp_id/id_comp` (`:142-157`).
- `pushforwardNatTrans`, `pushforwardCongr`: **GENUINELY ABSENT** at the presheaf level — grep over
  `Presheaf/` finds only `namespace PresheafOfModules`, no such decls; they exist ONLY at the sheaf
  level (`Sheaf/PushforwardContinuous.lean:154, 73`).
- **Existing presheaf adjunction `pullback φ ⊣ pushforward φ`: YES — `PresheafOfModules.pullbackPushforwardAdjunction`**
  (`Presheaf/Pullback.lean:50`, `= Adjunction.ofIsRightAdjoint (pushforward φ)`).
  **⇒ The project must NOT re-derive `pullback ⊣ pushforward`.** It is consumed as `adj2` in
  `leftAdjointUniq`. (Project comment at `:1124` already cites `Pullback.lean:44` correctly; the
  project already uses `pullbackPushforwardAdjunction` elsewhere — `QuotScheme.lean:416`,
  `Differentials.lean:52`.)
- **Gap**: divergent-with-cost only if the project re-derives the adjunction (it should not);
  the two absent helpers are a genuine (bounded) gap.
- **Verdict**: ALIGN_WITH_MATHLIB (reuse `pullbackPushforwardAdjunction`) + NEEDS_MATHLIB_GAP_FILL
  (port the two helpers).

### Decision 3: Cheapest path to `pushforward β ≅ pullback φ`

The `leftAdjointUniq` route IS the right idiom (`Adjunction.leftAdjointUniq`,
`CategoryTheory/Adjunction/Unique.lean:36`, signature `(adj1 : F ⊣ G) (adj2 : F' ⊣ G) : F ≅ F'`).
Minimal project-side decls (all sub-steps verified non-bottoming):

1. `PresheafOfModules.pushforwardNatTrans` — mirror `Sheaf/PushforwardContinuous.lean:154-167`,
   dropping `.val` (the sheaf version already operates on `X.val`/`.val.app U`/`X.val.presheaf.map`,
   i.e. the *presheaf* content). ~15-25 LOC. No new primitive.
2. `PresheafOfModules.pushforwardCongr` — mirror `:73-77`, via `PresheafOfModules.isoMk`
   (`Presheaf.lean:118`, app-wise iso ⇒ presheaf iso) + `ModuleCat.restrictScalarsCongr`
   (`ModuleCat/ChangeOfRings.lean:158`, EXISTS). ~6-10 LOC.
3. `PresheafOfModules.pushforwardPushforwardAdj` — mirror `:226-247` using presheaf `pushforwardId`/
   `pushforwardComp` + the two new helpers. ~40 LOC. Underlying `adj : F ⊣ G` for the open
   immersion is `f.isOpenEmbedding.isOpenMap.adjunction` (`IsOpenMap.adjunction`,
   `TopCat/Opens.lean:307`, EXISTS); `H₁`/`H₂` are the `:346-353` proofs ported verbatim.
4. `pushforward β ≅ pullback φ := (pushforwardPushforwardAdj …).leftAdjointUniq
   (pullbackPushforwardAdjunction φ.hom)` (or `Adjunction.leftAdjointUniq` applied to the two
   adjunctions). ~2 LOC.

Total ~70-90 LOC. **No sub-step is itself a Mathlib-absent primitive** — unlike the iter-214 `d.1`
failure (which needed genuinely new R_x-linear stalk infrastructure), every ingredient here is
either an existing decl or a mechanical de-sheafification of an existing sheaf-level decl that
already operates on presheaf data.

- **Verdict**: PROCEED (route correct) + NEEDS_MATHLIB_GAP_FILL (the three new decls, bounded).
