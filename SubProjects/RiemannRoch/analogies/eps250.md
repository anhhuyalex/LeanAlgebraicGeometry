## Headline answer to the one-shot question

**There is NO single sectionwise `ext`/`app`/coherence lemma that closes all of `(тИЧтИЧ)` at once, and
the project should stop hunting for one.** The obstruction is intrinsic, not a missing lemma: the LHS
contains the *sheafification unit* `presheafAdj.unit.app ЁЭЯЩс╡Ц` (= `CategoryTheory.toSheafify`, the
plus-construction map). `toSheafify.app X` is opaque тАФ it does NOT compute on sections тАФ so
`ext X; simp [app-lemmas]` cannot evaluate the LHS. Any route that survives must keep the unit
*abstract* until it is folded away.

The lemma that does fold it away **already exists and is already in hand**: the genuine "one-shot
coherence lemma" is `CategoryTheory.Adjunction.unit_app_unit_comp_map_╬╖`
(`Mathlib/CategoryTheory/Monoidal/Functor.lean:969`),
`adj.unit.app (ЁЭЯЩ_ C) тЙл G.map (╬╖ F) = ╬╡ G`, re-exported by the project as the CLOSED
`presheafUnit_comp_map_eta` (TensorObjSubstrate.lean:1502). It absorbs the opaque-unit part in one
move. What it does NOT also absorb is (i) the `.val`/pushforward reshaping and (ii) the
sheafification right-triangle that bring the LHS into the literal shape
`unit.app ЁЭЯЩс╡Ц тЙл (pushforward ╧Ж').map (╬╖ F)`; those are unavoidable bookkeeping. Only *after* that
collapse does the goal become `╬╡(pushforward ╧Ж') = (unitToPushforwardObjUnit ╧Ж).val`, which IS a
clean one-shot sectionwise `ext` (both sides `= ╧Ж.hom.app X`). So the correct shape is hybrid:
**abstract reshaping (i)+(ii) тЖТ the existing coherence lemma тЖТ sectionwise `ext` (iii)** тАФ exactly
the 3-step recipe already in the code, with the friction-defeating idioms below.

## Decisions identified

### Decision (i): `SheafOfModules.pushforward` on morphisms through `R_X = forget тЛЩ restrictScalars(ЁЭЯЩ)`

- **Mathlib idiom**: it is a **named `@[simp]` lemma**, not a defeq one must coax.
  `SheafOfModules.pushforward` is `@[simps map_val]`
  (`Mathlib/Algebra/Category/ModuleCat/Sheaf/PushforwardContinuous.lean:43`), generating
  `SheafOfModules.pushforward_map_val : ((pushforward ╧Ж).map f).val = (PresheafOfModules.pushforward ╧Ж.hom).map f.val`
  (verified via loogle, exact signature). Combine with `SheafOfModules.forget` `@[simps]`
  (`Mathlib/Algebra/Category/ModuleCat/Sheaf.lean:66`) тЗТ `forget_map : (forget R).map ╧Ж = ╧Ж.val`,
  `SheafOfModules.comp_val` (`Sheaf.lean:60`, `@[simp, reassoc]`, `(f тЙл g).val = f.val тЙл g.val`),
  and the `restrictScalars (ЁЭЯЩ)`-is-identity defeq (`restrictScalarsCompToPresheaf (ЁЭЯЩ) = Iso.refl`,
  `Presheaf/ChangeOfRings.lean:71`; project memory `restrictscalars-id-defeq`). Net rewrite:
  `R_X.map ((pushforward ╧Ж).map g) = (PresheafOfModules.pushforward ╧Ж').map g.val` holds by
  `simp only [Functor.comp_map, SheafOfModules.forget_map, SheafOfModules.pushforward_map_val]`
  with the `restrictScalars(ЁЭЯЩ)` step absorbed defeq.
- **Gap**: identical тАФ the named simp lemma exists. No rfl-coax needed.
- **Verdict**: PROCEED.

### Decision (ii): Y-side sheafification right-triangle in split `.val`/composite-functor form

- **Mathlib idiom**: it is `unit-naturality тЛЩ right-triangle`, two standard lemmas.
  `toSheafify_Y (F ЁЭЯЩс╡Ц) тЙл (a_Y.map (╬╖ F)).val` is the *unit naturality* of the presheaf
  sheafification adjunction at `╬╖ F`: the unit's underlying-presheaf component is `toSheafify`
  (`PresheafOfModules.toPresheaf_map_sheafificationAdjunction_unit_app`,
  `Mathlib/Algebra/Category/ModuleCat/Presheaf/Sheafification.lean:144`), so
  `(sheafificationAdjunction _).unit.naturality (╬╖ F)` gives
  `toSheafify (F ЁЭЯЩс╡Ц) тЙл (a_Y.map (╬╖ F)).val = ╬╖ F тЙл toSheafify ЁЭЯЩс╡Ц_Y`. Then `тЙл (sheafifyUnitIso.hom).val`
  is the **right triangle on the sheaf `ЁЭТк_Y`**: `sheafifyUnitIso = (asIso counit).app (unit Y)` and
  `ЁЭЯЩс╡Ц_Y = (unit Y).val`, so `toSheafify ЁЭЯЩс╡Ц_Y тЙл (sheafifyUnitIso.hom).val = ЁЭЯЩ` by
  `CategoryTheory.Adjunction.right_triangle_components`. Net: the composite `= ╬╖ F`.
- **Friction & idiom**: this needs ONE reassociation `t тЙл (A тЙл B) = (t тЙл A) тЙл B`, which is exactly
  where `rw [Category.assoc]` silently fails (the `тЙл` carries an implicit ring-presheaf argument that
  is `X.ringCatSheaf.val` vs `X.presheaf тЛЩ forgetтВВ тАж` тАФ defeq, not syntactic, so `rw`'s motive
  fails to unify). **Two robust idioms**: (a) reuse the project's proven X-triangle shape
  `(Category.assoc _ _ _).symm.trans (h тЦ╕ Category.id_comp _)` (it already works for `hXtri` at
  L1714); or (b) **`simp only [Category.assoc]`** rather than `rw` тАФ simp matches up to reducible
  defeq and reassociates where `rw` cannot. Best structural fix: open with a single
  `show тАж = тАж` that restates the goal in the syntactic `PresheafOfModules (X.presheaf тЛЩ forgetтВВ тАж)`
  category (since `X.ringCatSheaf.val` is `rfl`-equal to it), after which plain `rw [Category.assoc]`
  works for the rest of the block.
- **Gap**: identical тАФ both lemmas exist; reuse the L1714 shape (do NOT re-derive a new assoc path).
- **Verdict**: PROCEED.

### Decision (iii): STEP 7 тАФ `╬╡(PresheafOfModules.pushforward ╧Ж') = (unitToPushforwardObjUnit ╧Ж).val`

- **Mathlib characterization of the `╬╡`**: `PresheafOfModules.pushforward ╧Ж' = pushforwardтВАOfCommRingCat F R тЛЩ restrictScalars ╧Ж'`
  (`Presheaf/Pushforward.lean:86`). For a composite, `Functor.LaxMonoidal.comp` sets

  `╬╡(pushforward ╧Ж') = ╬╡(restrictScalars ╧Ж')` (the second factor is `G.map ЁЭЯЩ = ЁЭЯЩ`). The project's
  `restrictScalarsLax╬╡` (PresheafInternalHom.lean:290) gives the section value directly:
  `╬╡(restrictScalars ╧Ж').app X = Functor.LaxMonoidal.╬╡ (ModuleCat.restrictScalars (╧Ж'.app X).hom)`,
  and `ModuleCat.restrictScalars_╬╖` (`Mathlib/Algebra/Category/ModuleCat/Monoidal/Adjunction.lean:106`,

- **RHS section value**: `unitToPushforwardObjUnit_val_app_apply`

  `(unitToPushforwardObjUnit ╧Ж).val.app X a = ╧Ж.hom.app X a`. And `R_X.map upu` is defeq `upu.val`

- **Idiom**: a clean sectionwise `ext` тАФ `apply PresheafOfModules.hom_ext; intro X`

- **Gap**: identical тАФ every lemma exists; `epsilonPresheafToSheafUnit` is a thin project wrapper

- **Verdict**: PROCEED.
